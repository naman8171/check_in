# Disable Automatic Emails

This Odoo 18 Community module disables automatically triggered outgoing emails.

## What it blocks

- Email sends generated from `mail.template`.
- Email notifications generated by `mail.thread` followers and message notifications.
- Creation of `mail.mail` email records, including password reset and new-device alert emails.
- Direct SMTP hand-off through `ir.mail_server`.
- Queued outgoing mail processed by Odoo's mail queue.
- The standard Odoo mail scheduler is deactivated during module installation.

When disabled, Odoo returns an empty `mail.mail` recordset instead of creating default email records. Any pre-existing queued emails are moved to the `cancel` state instead of being sent, and direct SMTP sends return without delivering to the mail server.

## Settings toggle

Go to **Settings → General Settings → Email Delivery** and use **Disable Default Odoo Emails**.

The toggle is backed by the `disable_automatic_emails.enabled` system parameter, which defaults to `True`.
Turn the toggle off, or set the parameter to `False`, to restore normal email delivery.

Custom code can opt in to email delivery by using this context flag:

```python
record.with_context(disable_automatic_emails_bypass=True).message_post(...)
```

or:

```python
template.with_context(disable_automatic_emails_bypass=True).send_mail(record.id, force_send=True)
```
